# 极简数据获取指南

**适合技术新手的简单操作流程**

## 一、启动API服务（只需做一次）

1. **打开命令窗口**：
   - 按下 `Win + R` 键
   - 输入 `cmd`，按下回车键

2. **进入项目目录**：
   - 复制粘贴以下命令，按下回车键：
     ```
     cd c:\Users\agenew\Desktop\1\api
     ```

3. **启动服务**：
   - 复制粘贴以下命令，按下回车键：
     ```
     $env:PATH = "C:\Program Files\nodejs;$env:PATH"; node server.js
     ```

4. **验证服务启动**：
   - 看到窗口显示 "文章同步API服务已启动，监听端口 3000" 就成功了
   - 这个窗口要保持打开状态，不要关闭

## 二、获取真实数据（每次需要时操作）

### 方法1：使用浏览器查看管理后台（推荐）

1. **打开管理后台**：
   - 找到桌面上的 `admin.html` 文件
   - 双击打开它

2. **查看数据**：
   - 页面会自动加载真实数据
   - 顶部显示总文章数、总浏览量、总点赞数等统计信息
   - 下方显示热门文章排行榜
   - 页面每30秒会自动刷新一次数据

3. **手动刷新数据**：
   - 点击页面底部的"刷新数据"按钮
   - 或按键盘上的 `F5` 键

### 方法2：直接访问API获取数据（进阶）

1. **打开浏览器**：
   - 打开任意浏览器（Chrome、Edge、Firefox等）

2. **访问文章列表**：
   - 在地址栏输入：`http://localhost:3000/api/v1/articles`
   - 按下回车键
   - 页面会显示所有文章的JSON数据

3. **访问健康检查**：
   - 在地址栏输入：`http://localhost:3000/api/v1/health`
   - 按下回车键
   - 页面会显示API服务的健康状态

## 三、常见问题解决

### 问题1：管理后台显示数据为0
- **解决方法**：确保API服务窗口是打开的，然后按 `F5` 刷新页面

### 问题2：无法打开管理后台
- **解决方法**：确认文件路径正确，尝试使用浏览器打开（右键文件，选择"打开方式" → 选择浏览器）

### 问题3：API服务启动失败
- **解决方法**：检查命令窗口的错误信息，确认Node.js已安装

### 问题4：刷新后数据不变
- **解决方法**：确保浏览器没有缓存，尝试使用Ctrl+F5强制刷新

## 四、数据说明

### 数据来源
- **真实数据**：来自API服务，包含最新的浏览量和点赞数
- **同步机制**：前端页面会自动向API发送同步请求
- **存储位置**：数据保存在SQLite数据库中

### 数据范围
- **文章总数**：7篇
- **分类**：4个（命学基础、名人八字、花果论心法、杂谈随笔）
- **数据字段**：标题、内容、浏览量、点赞数、日期等

## 五、注意事项

1. **不要关闭API服务窗口**：关闭窗口会导致服务停止
2. **定期备份数据**：重要数据请定期备份
3. **安全注意事项**：
   - 不要在公共网络上暴露API服务
   - 生产环境需要修改默认密码
4. **浏览器兼容性**：推荐使用Chrome或Edge浏览器

## 六、技术支持

如果遇到其他问题：
1. 查看API服务窗口的错误信息
2. 检查浏览器控制台日志（按F12打开）
3. 重启API服务和浏览器
