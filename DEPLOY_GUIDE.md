# 一键部署使用指南

专为小白用户设计的详细部署教程

## 什么是一键部署？

一键部署是一个简单的脚本，让你只需要点击或输入一个命令，就能自动完成网站的部署工作，不需要记住复杂的Git命令。

## 系统要求

- Windows 系统
- 已经安装了 Git
- 已经配置了 GitHub 仓库

## 部署脚本说明

我为你创建了两种部署脚本，你可以选择其中一种使用：

1. **deploy.bat** - 适合命令提示符用户
2. **deploy.ps1** - 适合 PowerShell 用户

## 方法一：使用命令提示符（推荐小白用户）

### 步骤1：打开项目文件夹

1. 找到你的项目文件夹（通常在桌面上，名为 `1`）
2. 按住 `Shift` 键，同时右键点击空白处
3. 选择 **在此处打开命令窗口** 或 **在此处打开 PowerShell 窗口**

### 步骤2：运行部署脚本

在打开的命令窗口中，输入以下命令并按 `Enter`：

```cmd
.deploy.bat
```

### 步骤3：查看部署结果

- 如果看到 `部署成功！`，说明部署完成
- 如果看到 `部署失败！`，请检查错误信息

## 方法二：使用 PowerShell

### 步骤1：打开 PowerShell

1. 点击 `开始` 菜单
2. 输入 `PowerShell` 并打开
3. 输入 `cd C:\Users\agenew\Desktop\1` 并按 `Enter`（进入项目文件夹）

### 步骤2：运行部署脚本

在 PowerShell 中输入以下命令并按 `Enter`：

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\deploy.ps1
```

### 步骤3：查看部署结果

- 如果看到 `Deploy successful!`，说明部署完成
- 如果看到 `Deploy failed!`，请检查错误信息

## 脚本会自动做什么？

1. **检查状态**：查看当前 Git 状态
2. **自动提交**：如果有未提交的更改，自动提交
3. **推送代码**：将代码推送到 GitHub
4. **触发部署**：GitHub Actions 自动部署到 Pages
5. **显示结果**：告诉你部署是否成功

## 部署成功的标志

- 命令窗口显示成功信息
- GitHub 上能看到你的最新代码
- 网站会在几分钟后自动更新

## 常见问题及解决方法

### 问题1：找不到脚本文件
**解决方法**：确保你在项目文件夹中运行命令，命令前面的 `.` 不能省略。

### 问题2：脚本运行被阻止
**解决方法**：使用 PowerShell 命令时，确保包含 `-ExecutionPolicy Bypass` 参数。

### 问题3：部署失败
**解决方法**：
1. 检查网络连接
2. 检查 GitHub 账号是否登录
3. 查看命令窗口中的错误信息

## 部署时间

- 脚本运行时间：通常几秒钟
- 网站更新时间：GitHub Actions 部署需要 1-5 分钟

## 注意事项

1. 运行脚本前，请确保你的代码已经完成修改
2. 脚本会自动提交所有未提交的更改
3. 部署成功后，网站会自动更新，不需要手动操作
4. 如果遇到问题，可以重新运行脚本

## 总结

1. 打开项目文件夹
2. 打开命令窗口
3. 运行 `.deploy.bat` 或 PowerShell 命令
4. 等待部署完成
5. 查看部署结果

恭喜！你已经学会了如何使用一键部署脚本，现在可以轻松部署你的网站了！
